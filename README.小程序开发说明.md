# 小程序开发说明

## 🚨 重要：如何正确打开项目

### 问题现象
如果在小程序开发工具中看到错误：
- "在项目根目录未找到 app.json"
- "app.json 文件内容错误"

### 根本原因
本项目使用 **Taro 框架**开发，需要编译后才能在小程序开发工具中运行。

### ✅ 正确的打开方式

#### 方法一：直接打开 dist 目录（推荐）

1. **先构建项目**
   ```bash
   npm run build:weapp
   ```

2. **在小程序开发工具中打开 `dist` 目录**
   - 文件路径：`d:\google_code\note\local\daohen-weapp_V5\daohen-weapp\dist`
   - **不要**打开项目根目录！

3. **编译并运行**
   - 小程序开发工具会自动识别 `app.json` 文件
   - 点击"编译"按钮即可运行

#### 方法二：修改项目配置

如果一定要在项目根目录打开，需要：

1. 在项目根目录创建 `app.json` 文件（不推荐，会导致配置混乱）

## 📝 开发流程

### 日常开发步骤

1. **修改源代码**
   - 源文件位置：`src/` 目录
   - 修改 `.tsx`、`.ts`、`.scss` 等文件

2. **实时编译**
   ```bash
   npm run dev:weapp
   ```
   这会启动监听模式，自动编译代码

3. **小程序开发工具自动刷新**
   - 确保小程序开发工具开启了"编译后自动刷新"
   - 或者手动点击"编译"按钮

### 生产构建

```bash
npm run build:weapp
```

## ⚠️ 常见错误及解决方案

### 错误 1：未找到 app.json

**原因**：在项目根目录打开，而不是 `dist/` 目录

**解决**：
1. 运行 `npm run build:weapp`
2. 在小程序开发工具中打开 `dist/` 目录

### 错误 2：app.json 格式错误

**原因**：Taro 构建的 JSON 文件被压缩

**解决**：
1. 重新运行 `npm run build:weapp`
2. 或者手动格式化 `dist/app.json` 文件

### 错误 3：云开发环境不可用

**原因**：云开发环境 ID 配置错误或网络问题

**解决**：
1. 检查 `config/dev.ts` 或 `config/prod.ts` 中的云开发环境配置
2. 确保网络连接正常
3. 在小程序开发工具中重新登录微信账号
