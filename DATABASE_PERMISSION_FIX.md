# 数据库权限问题解决方案

## 问题现象
```
errCode: -502003 database permission denied | errMsg: Permission denied
```

## 根因分析
根据腾讯云开发文档，数据库进行读写时会以 `_openid` 字段作为数据归属判定依据。当前数据库集合的权限设置不允许用户进行读写操作。

## 解决方案（按推荐顺序）

### 方法一：在微信开发者工具中配置（最快，推荐）

#### 步骤 1：打开云开发控制台
1. 打开 **微信开发者工具**
2. 点击左侧工具栏的 **"云开发"** 按钮（云朵图标）
3. 进入云开发控制台

#### 步骤 2：配置 entries 集合权限
1. 点击 **"数据库"** 标签
2. 找到并点击 `entries` 集合
3. 点击 **"权限设置"** 按钮
4. 选择以下权限配置之一：

**选项 A：快速配置（推荐用于开发测试）**
- 权限模式：**所有用户可读，仅创建者可写**
- 点击"保存"

**选项 B：安全配置（推荐用于生产环境）**
- 权限模式：**自定义安全规则**
- 输入以下规则：
```json
{
  "read": "auth.openid != null",
  "write": "auth.openid != null"
}
```
- 点击"保存"

#### 步骤 3：配置 users 集合权限
1. 在数据库列表中找到并点击 `users` 集合
2. 点击 **"权限设置"** 按钮
3. 选择：**所有用户可读，仅创建者可写**
4. 点击"保存"

### 方法二：创建索引优化查询（性能优化）

根据控制台提示，为 `entries` 集合创建索引：

1. **在微信开发者工具中打开云开发控制台**
2. **进入数据库 → 索引管理**
3. **创建以下索引：**
   - 集合：`entries`
   - 字段：`createTime`
   - 排序：**降序**
   - 索引名称：`createTime_desc`
   - 点击"创建"

4. **为 `users` 集合创建索引：**
   - 集合：`users`
   - 字段：`_openid`
   - 排序：**升序**
   - 索引名称：`openid_asc`
   - 点击"创建"

### 方法三：检查云开发环境

1. **确认已开通云开发**
   - 在微信开发者工具中，点击"云开发"
   - 如果未开通，按照提示开通云开发服务

2. **检查环境 ID 是否正确**
   - 在云开发控制台，查看当前环境 ID
   - 确认与代码中配置的环境 ID 一致：`n1-daohen-9ga3v5k246792d59`

3. **确认已创建数据库集合**
   - 进入数据库页面
   - 确认已创建 `entries` 和 `users` 两个集合
   - 如果没有，手动创建

## 快速操作步骤（5 分钟搞定）

1. 在微信开发者工具中按 `Ctrl+Shift+P` (Windows) 或 `Cmd+Shift+P` (Mac)
2. 输入 "云开发" 并选择
3. 进入数据库页面
4. 点击 `entries` 集合 → 权限设置 → 选择 **"所有用户可读，仅创建者可写"** → 保存
5. 点击 `users` 集合 → 权限设置 → 选择 **"所有用户可读，仅创建者可写"** → 保存
6. 返回小程序，重新编译测试

## 验证修复

修复后，执行以下操作验证：

1. **重新编译小程序**
   - 在微信开发者工具中点击"编译"按钮

2. **测试添加记录**
   - 点击"+"按钮添加一条新的道痕记录
   - 填写完整信息并保存
   - 应该不再出现权限错误

3. **测试编辑记录**
   - 点击已创建的记录
   - 点击"编辑"按钮
   - 修改内容并保存
   - 应该成功保存

4. **测试删除记录**
   - 点击已创建的记录
   - 点击"删除"按钮
   - 确认删除
   - 应该成功删除

## 常见问题

### Q1: 找不到"权限设置"按钮？
**A**: 确保您使用的是最新版本的微信开发者工具，并且已开通云开发服务。

### Q2: 权限配置后仍然报错？
**A**: 
1. 检查数据库集合中是否有 `_openid` 字段
2. 确保用户已登录，可以获取到 openid
3. 尝试清除小程序缓存后重新编译

### Q3: 生产环境应该使用什么权限配置？
**A**: 建议使用自定义安全规则：
```json
{
  "read": "doc._openid == auth.openid",
  "write": "doc._openid == auth.openid"
}
```
这样可以确保用户只能访问自己的数据。
