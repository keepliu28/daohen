### ✅ 6. 数据库权限问题（已解决）
- **错误**：`-502003 database permission denied`
- **现象**：新建笔记时报错，云数据库为空，但本地缓存有数据
- **原因**：云开发数据库集合权限未正确配置
- **根本问题**：错误处理逻辑缺陷，权限错误时仍返回"成功"，误导用户
- **解决方案**：
  1. 配置数据库权限（entries 和 users 集合）
  2. 修复错误处理逻辑，明确区分权限错误
  3. 权限错误时抛出明确错误，不再误导用户
- **修复文件**：`src/utils/storage.ts`
- **修复方法**：
  - 权限错误（-502003）：抛出明确错误
  - 其他错误：降级到本地保存，但标记为降级
- **配置步骤**：
  1. 微信开发者工具 → 云开发 → 数据库
  2. entries 集合 → 权限设置 → 自定义权限
  3. 读取权限：`true`，写入权限：`auth.uid != null`
  4. 同样配置 users 集合
- **验证结果**：✅ 基本功能跑通，数据正常保存到云数据库
